│ Plan to implement                                                            │
│                                                                              │
│ Plan: PowerPoint (.pptx) and OpenDocument (.odp) Import/Export               │
│                                                                              │
│ Context                                                                      │
│                                                                              │
│ The app currently only exports to PDF/PNG and imports from its own JSON      │
│ format. Users need to exchange presentations with PowerPoint and             │
│ LibreOffice/OpenOffice users. This adds import and export support for .pptx  │
│ (PowerPoint) and .odp (OpenDocument Presentation) formats.                   │
│                                                                              │
│ Coordinate System                                                            │
│                                                                              │
│ - App: 960×540px (16:9). At 96 DPI → 10" × 5.625"                            │
│ - PPTX: EMUs (1 inch = 914400 EMU). Slide = 9144000 × 5143500 EMU (10" ×     │
│ 5.625")                                                                      │
│ - ODP: centimeters. Slide = 25.4 × 14.2875 cm                                │
│                                                                              │
│ Conversion: px * (914400 / 96) for PPTX EMU, px * (2.54 / 96) for ODP cm.    │
│                                                                              │
│ Dependencies to install                                                      │
│                                                                              │
│ - pptxgenjs — PPTX generation (handles all OOXML complexity)                 │
│ - jszip — ZIP reading for PPTX/ODP import, and ODP export (ODP is a ZIP of   │
│ XML files)                                                                   │
│                                                                              │
│ Changes                                                                      │
│                                                                              │
│ 1. Install dependencies                                                      │
│                                                                              │
│ npm install pptxgenjs jszip                                                  │
│                                                                              │
│ 2. Create src/utils/exportPptx.ts — PPTX export                              │
│                                                                              │
│ Uses pptxgenjs which provides a high-level API:                              │
│                                                                              │
│ export async function exportPptx(presentation: Presentation): Promise<void>  │
│                                                                              │
│ Mapping:                                                                     │
│ - Slide backgrounds → slide.background (solid color, gradient, image)        │
│ - TextElement → slide.addText() with formatting options (font, size, bold,   │
│ italic, color, align)                                                        │
│   - Parse markdown headers (# ## ###) into font size multipliers             │
│   - Parse bullet lists (- *) into pptxgenjs bullet format                    │
│ - ShapeElement → slide.addShape() for rect/ellipse/triangle/star,            │
│ slide.addShape(LINE) for line/arrow                                          │
│   - Map fill, stroke, strokeWidth, cornerRadius, rotation, opacity           │
│ - ImageElement → slide.addImage() with base64 data from resources            │
│   - Apply crop parameters                                                    │
│ - Speaker notes → slide.addNotes()                                           │
│ - Element rotation → rotate option (degrees)                                 │
│ - Element opacity → pptxgenjs transparency (100 - opacity*100)               │
│                                                                              │
│ Coordinate conversion: px / 96 → inches (pptxgenjs uses inches)              │
│                                                                              │
│ 3. Create src/utils/importPptx.ts — PPTX import                              │
│                                                                              │
│ Uses jszip to unzip and DOMParser to parse OOXML XML:                        │
│                                                                              │
│ export async function importPptx(file: File): Promise<Presentation>          │
│                                                                              │
│ Process:                                                                     │
│ 1. Read file as ArrayBuffer, open with JSZip                                 │
│ 2. Parse ppt/presentation.xml for slide list and slide size                  │
│ 3. For each slide in ppt/slides/slideN.xml:                                  │
│   - Parse p:cSld/p:bg for background (solid fill, gradient)                  │
│   - Parse p:cSld/p:spTree for shapes:                                        │
│       - p:sp (shapes with text) → TextElement or ShapeElement                │
│     - p:pic (pictures) → ImageElement                                        │
│   - Parse p:txBody for text content with runs (a:r) and formatting (a:rPr)   │
│   - Extract position from a:off (EMU) and size from a:ext (EMU)              │
│   - Convert EMU → px: emu / (914400 / 96)                                    │
│ 4. Extract media files from ppt/media/ → create Resources with data URLs     │
│ 5. Parse _rels files for media relationships                                 │
│ 6. Parse speaker notes from ppt/notesSlides/                                 │
│ 7. Assemble into Presentation structure using createSlide(),                 │
│ createTextElement(), etc.                                                    │
│                                                                              │
│ 4. Create src/utils/exportOdp.ts — ODP export                                │
│                                                                              │
│ Constructs ODF ZIP manually with JSZip:                                      │
│                                                                              │
│ export async function exportOdp(presentation: Presentation): Promise<void>   │
│                                                                              │
│ ODP ZIP structure:                                                           │
│ - mimetype — application/vnd.oasis.opendocument.presentation (uncompressed,  │
│ first entry)                                                                 │
│ - META-INF/manifest.xml — file manifest                                      │
│ - content.xml — slides, elements, styles                                     │
│ - styles.xml — master page styles                                            │
│ - Pictures/imageN.ext — embedded images                                      │
│                                                                              │
│ Element mapping to ODF XML:                                                  │
│ - TextElement → <draw:frame><draw:text-box><text:p> with <text:span> for     │
│ formatting                                                                   │
│ - ShapeElement → <draw:rect>, <draw:ellipse>, <draw:line>,                   │
│ <draw:custom-shape>                                                          │
│ - ImageElement → <draw:frame><draw:image xlink:href="Pictures/..."/>         │
│ - Background → <draw:fill-image> or <style:drawing-page-properties           │
│ draw:fill="solid">                                                           │
│                                                                              │
│ Coordinate conversion: px * (2.54 / 96) → cm string with "cm" suffix         │
│                                                                              │
│ 5. Create src/utils/importOdp.ts — ODP import                                │
│                                                                              │
│ export async function importOdp(file: File): Promise<Presentation>           │
│                                                                              │
│ Process:                                                                     │
│ 1. Read as ArrayBuffer, open with JSZip                                      │
│ 2. Parse content.xml with DOMParser                                          │
│ 3. For each <draw:page>:                                                     │
│   - Extract background from drawing-page-properties                          │
│   - For each <draw:frame>, <draw:rect>, <draw:ellipse>, etc.:                │
│       - Read position/size from svg:x, svg:y, svg:width, svg:height          │
│ attributes                                                                   │
│     - Convert cm → px: parseFloat(value) / 2.54 * 96                         │
│     - Identify text frames, images, shapes                                   │
│   - Extract text content and formatting from <text:p> and <text:span>        │
│ 4. Extract images from Pictures/ folder → create Resources                   │
│ 5. Assemble into Presentation using factory functions                        │

● Build passes. Here's a summary of the three root causes and fixes:

  Root cause 1: Elements at top-left corner — PowerPoint shapes often have no
  <a:xfrm> in their spPr because they're placeholders (title, subtitle, body)
  that inherit position from the slide layout. The old code returned {x:0, y:0, 
  width:200, height:150} defaults. Now the importer:
  - Parses the actual slide layout files to get placeholder positions
  - Falls back to sensible hardcoded defaults for standard placeholder types
  - Skips shapes with no position info at all instead of placing them at 0,0

  Root cause 2: Elements scaled 10-20x — PowerPoint uses group shapes
  (<p:grpSp>) with their own child coordinate space. Elements inside groups have
   positions relative to the group's internal coordinate system. The old code
  used getElementsByTagNameNS (recursive), which found shapes inside groups but
  read their group-relative coordinates as slide coordinates. Now the importer:
  - Uses directChildren() instead of recursive getElementsByTagNameNS to only
  process top-level shapes
  - Has a dedicated processGroupShape() that correctly transforms child
  coordinates through the group's chOff/chExt → off/ext mapping
  - Handles nested groups recursively

  Root cause 3: Slide size mismatch — PowerPoint presentations can have any
  slide size (not just 16:9). The old code assumed coordinates were already in
  960×540 space. Now it reads the actual <p:sldSz> from presentation.xml and
  scales all coordinates to fit our canvas.
